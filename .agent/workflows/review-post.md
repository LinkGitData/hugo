description: 專門用來單獨檢視某篇文章或者現在正在處理的文章的正確性、Markdown 格式以及圖片連結是否正確，並自動執行修正。
---

當使用者呼叫此 Workflow (例如輸入 `/review-post`，或是 `/review-post [檔名]`) 時，請嚴格執行以下文章除錯與修復任務：

### 1. 鎖定目標文章
* 如果使用者有提供特定檔名，請在 `link-blog/content/posts/` 目錄下尋找該文章。
* 如果使用者未提供檔名，請自動鎖定「當下正在處理的」或「最近被修改過」的 Markdown 文章。

### 2. 結構與 Frontmatter 稽核
* **格式確認**：檢查文章最前方的 YAML Frontmatter 是否包含完整的必填欄位 (`title`, `date`, `draft`, `tags`, `categories`, `cover`)。
* **資料來源檢查**：確保文章的開頭（Frontmatter 結束後的第一段）有使用引言區塊標示官方來源（如 `> 🔗 **官方資料來源**：`）。若缺少此結構，請提醒使用者。

### 3. Markdown 格式與粗體防呆稽核 (極度重要)
針對 Hugo Goldmark 解析器容易破版的特性，請仔細掃描內文的每一個 `**` 粗體標籤，並**主動修復**任何錯誤：
* ❌ **內側不可有空白**：搜尋是否有類似 `** 錯誤的文字 **` 的寫法，強制將內側的空白移除，改為 `**正確的文字**`。星號必須緊貼內部文字。
* ❌ **外側必須有空白**：搜尋粗體標籤若緊貼著中文字或「全形標點符號」（如 `「`、`」`、`（`、`）`、`，`、`。`），其**外側**必須加上一個半形空格。強制將 `這是**關鍵字**。` 修改為 `這是 **關鍵字** 。`。

### 4. 視覺圖片與尺寸稽核
* **圖片連結有效性**：萃取文章內所有的圖片連結（包含 Frontmatter 的 `cover.image` 以及內文的 `![]()`），並透過 `list_dir` 或腳本檢查該圖片是否真實存在於 `link-blog/static/images/` 資料夾中。
* **封面尺寸自動修復**：如果發現封面圖片，請檢查其長寬比是否符合首頁卡片標準的 `1024x256`。如果不符合，請直接撰寫並執行 Python PIL 腳本，進行「置中、等比縮放並裁切」，並覆寫原圖。

### 5. Mermaid 語法與容器稽核
如果文章內包含流程圖：
* **短代碼檢查**：確保圖表被包覆在 `{{< mermaid >}}` 與 `{{< /mermaid >}}` 之間，**絕對不可**使用原生的 ` ```mermaid ` 區塊。
* **節點文字防呆**：檢查 Mermaid 節點名稱中是否混入了會導致舊版解析器崩潰的特殊符號（例如引號 `""`、括號 `()`、斜線 `/` 等）。若發現，請自動將其移除，只保留純中英文與數字。

### 6. 自動修復與回報 (Self-Healing)
* **主動修改**：發現上述任何錯誤後，Agent 必須運用 `multi_replace_file_content` 工具或是撰寫 Python 腳本，**直接對原始 Markdown 檔案進行修改與覆寫**。
* **版控同步**：修復完成後，自動執行 `git add` 與 `git commit -m "fix(blog): auto-review formatting and structure"`，並 Push 至遠端倉庫。
* **總結回報**：最後向使用者列出「剛才發現並修復了哪些問題」（例如：修復了 3 處粗體空白錯誤、裁切了封面圖片）。
